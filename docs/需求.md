# 專案需求規格書 (Requirements Specification)
思考、回復、文件、計畫、專案結構及相關文件，盡可能都用繁體中文描述。

## 1. 專案概述 (Project Overview)
**專案名稱**：dont_sleep (別睡覺)
**專案目標**：因應工作需求，開發一個能防止電腦進入螢幕保護程式或睡眠狀態的工具程式。
**技術堆疊**：
- 程式語言：C#
- 輸出格式：Windows 可執行檔 (.exe)

---

## 2. 功能需求 (Functional Requirements)

### 2.1. 運作模式
程式需支援兩種主要模式，使用者可進行切換設定：

#### 2.1.1. 醒者模式 (Keep Awake)
- 保持系統喚醒，防止進入睡眠或螢幕保護程式。

#### 2.1.2. 模擬螢幕保護模式 (Simulate Screensaver)
- **預設狀態**：此為程式啟動時的預設模式。
- **設定功能**：
  - 可設定「進入模擬時間」(閒置多久後啟動)，**預設值：3分鐘**。
  - 可設定「喚醒密碼」，**預設值：`** (Backtick)。
- **運作行為**：
  - 觸發時直接讓螢幕進入休眠模式 (Turn off monitor)。
- **解鎖機制**：
  - 在模擬螢幕保護運作期間，持續監控鍵盤輸入。
  - 使用者無需叫出輸入視窗，直接鍵入正確密碼即可解除模擬模式。

### 2.2. 使用者介面與資訊顯示 (UI/UX)
- **視覺風格**：採暗色系設計，背景使用黑色。
- **硬體資源監控**：
  - 顯示 CPU、RAM、DISK (磁碟) 使用量。
  - **預設顏色**：亮綠色。
  - **警示顏色機制**：
    - **RAM (記憶體)**：
      - 使用量 > 70%：顯示為**亮橘色**。
      - 發生 SWAP (置換) 時：顯示為**亮紅色** (或使用量 > 85% 時)。
    - **DISK (磁碟)** & **RAM (記憶體)** 區間警示：
      - 70% ~ 85%：顯示為**亮橘色**。
      - \> 85%：顯示為**亮紅色**。

- **詳細資訊顯示**：
  - 列出記憶體使用量前 3 名的程序。
  - 顯示格式：`<RAM使用量>` `<程式名稱+(執行緒數)>` (參照工作管理員)。
  - 排版要求：資料需對齊排列，便於閱讀。

### 2.3. 系統列整合 (System Tray / Notification Area)
- **縮小功能**：視窗可最小化至 Windows 通知區域 (System Tray)，不出現在工作列。
- **即時資訊提示 (Tooltip)**：
  - 當滑鼠游標移至通知區域圖示上方時，需顯示當前資源使用量。
  - 格式範例：`C35% R59% D50%`。
